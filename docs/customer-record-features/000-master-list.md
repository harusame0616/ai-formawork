# 顧客カルテシステム機能マスターリスト

## 優先度分類基準

- **P0 (Critical)**: システムの核心機能。これがないとサービスが成立しない
- **P1 (High)**: ビジネス価値が高く、多くのユーザーが必要とする機能
- **P2 (Medium)**: 利便性を向上させる機能。差別化要因になる
- **P3 (Low)**: あると便利だが、必須ではない機能

## 次のステップ

各機能について以下を含む詳細ドキュメントを作成します：

1. **機能概要**: なぜこの機能が必要なのか
2. **ユースケース**: どのようなときに役立つのか
3. **重要度評価**: どれぐらい重要か、なぜそう判断したのか
4. **基本設計**: システム構成、データ構造、フロー
5. **詳細設計**: API仕様、UI/UX、実装詳細、テスト方針

各機能は `001-feature-name.md` から `425-feature-name.md` まで個別のファイルとして作成されます。
